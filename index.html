About

I’m a Unity Developer with a bit over 5 years in mobile F2P, mostly working on gameplay systems and UI/VFX flows. I focus on clear feedback, a responsive feel, and LiveOps that stays stable when lots of things are happening. I like connecting core systems, UI sequencing, and backend events so reward screens, animations, and data all stay in sync. I’ve co-developed playable prototypes, tuned timings and input feel to reduce perceived latency, and cut reward-flow conflicts by using queue-based orchestration. I’ve also improved layouts and accessibility across devices, and wired event/reward screens into REST/JSON endpoints with analytics hooks so what players see matches what the backend records. I usually work with Unity C#, DOTween, Addressables, Animator, UGUI/UI Toolkit, Cinemachine, Shader Graph, Git, and when it fits the project, Nakama/Firebase/PlayFab. I’m aiming for Gameplay Programmer / Unity Developer roles (remote or relocation), where I can ship clean, reliable features and keep pushing the feel and clarity of the game.

Beast Battle - Unreal Engine 4


Beast Battle is a project where I have been experimenting to develop a turn-based battle mechanics with a top view camera inspired by the game Diablo 4.


Mechanics Log

Interacting with the UI and click on the button to spawn an enemy at the field and start the game.

Clicking on the button the main character generates the attack animation and the enemy show another animation when receive a hit.

When the potion button is clicked, the main character restore some part of the health.

The special attack may failed, when it does, shows a message that the attack has been failed.

Shows the death animation for the main character when the health bar is on zero points.

Shows the death animation for the enemy, the game ends and appears a restart button.

When the restart button is clicked, the game start again.


Wizard Runner - Unreal Engine 4


Wizard Runner is a project inspired on a subgenre of platform game, trying to avoid the obstacles and moving forward the most of you can.


Mechanics Log

When the character is moving on and hit a box collision, a new floor spawn on the field, generating an infinite floor.

Adding an animation blueprint to an imported skeletal mesh, allows to move the characters with the WASD keys and the space bar to jump.

This mechanic add random obstacle while the character is moving forward, when the character hits an obstacle, trigger an event who disappear the character and replace with an explosion animation.


Marble Run - Unreal Engine 4


Marble Run is a simple project that consist to rotate a platform and try to take out the ball inside the maze.


Mechanics Log

This mechanic gets a mouse X,Y inputs to set an actor rotation to rotate the maze.

This mechanic activate the win condition when the ball hits a trigger box when is escaping the maze.
